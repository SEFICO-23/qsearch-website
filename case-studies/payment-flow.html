The Engagement

Systematic research on a payment processor handling billions daily

A Tier-1 global payment processor with billions of daily transactions maintains a public security research program through a leading bug bounty platform. QuantumSearch conducted systematic research into the payment flow integrity of their APIs, focusing on invoice processing, order management, and payment routing.

The research was documentation-driven: before sending a single request, we mapped the entire API surface across 9 API families, 30+ GraphQL mutations, and 5 acquisition-integrated payment rails. Findings emerged from reading the documentation like source code and identifying logical inconsistencies.

What We Found

Payment routing and invoice manipulation at scale

10+ confirmed findings across the payment API surface, spanning invoice manipulation, payment routing integrity, and information disclosure.

Lead Finding Invoice Status Manipulation Medium

The invoicing API accepted status transitions that bypassed the intended payment verification flow. An attacker could mark invoices as paid without completing the actual payment transaction, generating fraudulent QR-based payment receipts indistinguishable from legitimate ones.

Supporting Finding Payment Routing Inconsistency Critical

The order management API permitted modification of payment recipient details on buyer-approved transactions. After a buyer authorized a payment to Merchant A, the API accepted a modification redirecting funds to Merchant B without requiring buyer re-authorization. This finding was independently discovered and confirmed as a known issue being actively addressed.

Methodology

Documentation-first, precision-driven

We don't spray requests hoping for errors. We read API surfaces like source code and find the logical inconsistencies that scanners miss entirely.

  • Full API surface mapping from public documentation (9 API families, 200+ endpoints)
  • Pattern analysis: identifying every endpoint where post-authorization mutation is possible
  • Cross-platform pattern matching: testing the same logical vulnerability class across multiple payment processors
  • Sandbox credential acquisition and controlled testing in non-production environment
  • Conservative severity assessments (never overclaiming impact)
Business Impact

Payment integrity at scale is a trust problem

Payment flow integrity vulnerabilities in a processor handling billions in daily volume expose merchants and buyers to financial fraud. Invoice manipulation creates trust risks for businesses relying on the platform's payment confirmation as proof of payment.

Outcome

Bounty-paid. Methodology validated.

  • Finding submitted and paid through the platform's security program
  • Research methodology validated: documentation-first, precision-driven
  • Cross-platform pattern analysis identified the same vulnerability class in multiple payment processors
